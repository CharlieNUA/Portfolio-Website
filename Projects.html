Laser Beam

Download Page Documentation

Laser Beam is a top down puzzle game that makes the player use light reflections to hit the targets and progress to the next level, the player also has the added difficultly of obstacles that might get in the way or tricky movements to work around. There are 20 levels in the game that add new puzzles for the player to work on top of this the player has to complete this before the timer runs out.There is a good amount of customisation in this game as the player can now change the color beam to any color of choice.

Engine & Tools: Unreal engine 4, Rider 2019, Github, Photoshop

Laser Beam Overview

Originaly the design was a water management game that used a block to reflect water into a goal, however, after working with the Unreals Niagara system I thought this would take too long for the time we had set for our project. So instead this was turned into a laser beam game using Unreals line trace and calculating the reflection on mirror objects. Which I worked on as a solo project to get fully finished and playable with 20 levels.
Laser Beams main focus was to be a puzzle game so alot of the time and research was spent on learning how to make things more difficult and what makes levels harder than others, so i could then have a progressing game that became more and more difficult as the player progresses but also teaches the player new skills that feel rewarding later on.

Basic Mechanics



The main movement for the cube is sorted out through a variable which can be effected by pressing either A or D, which will then rotate the cube either clockwise or anti-clockwise depending on what button is pressed. The linetrace is stored inside the cube which will draw a line between the cube and the edges of the map unless the line trace hits a perticular material which in this case its the glass material and that then spawns in another line trace and so forth.



The linetrace can then detect wether its hitting another reflective material in which it will spawn another that has the same behaviour, if the line trace doesn't detect that exact material name it will just draw the black line.This indicates that the line trace has ended and its not hit a target. To calculate the angle of the other light reflection the trace will then take into account the direction its coming from and then reflect it on a oppersite angle so if the beam is going down right it will then reflect up right as the picture shows above.

Space Cat Nip

Download Page Documentation

Made for the Global Games Jam 2022, I worked on this with another Developer and an Artist to create a third-person shooter classic sci-fi over a couple of days. My main focus on this game was the robot enemies that chase the player around the map and also Spent a lot of time on the gravity effect to make all the items and islands float which then would react to the player's movement.

Engine & Tools: Unreal engine 4, Visual Studio 2017, Github

Space Cat Nip Overview

Space Cat Nip is mainly just a simple platformer with some added extra gravity effects that make it alot of fun for the player to jump around on floating islands and push rocks around the map.The main movement is done through UE4 character controller so WASD will move the character and space will make it jump, after aspect we spent a good amount of time on was the UE4 character components air movement as with this we could then give the player character space like movement so the character would be able to control alot in the air and the gravity would be very small falling when the character jumps of a rock making it easy to land on small ones.
Since this was intended for a games jam the time we had to work on it was only two days so the scope had to be quite small, however it still includes a couple of basic mechanics which makes it fun to play and go through the levels.With the robot enemies that show up later on the game does then have some small difficulty as the robots can then chase the player down.

AI Behaviour



The Robot enemies where a idea we came up with later on in the jam so they didn't have much time for us to develop on however with the UE4 AI behaviour trees we could get a simple patroling and chasing player implemented into the game.So the Robots behaviour will make it find a location at random on the nav mesh its using so for this it would be the islands top face, and then move to it which it then can repeat the process.If the enemy detects the player in its line of sight it can set the players position as the target and then move to that.As we needed a way of dying the player takes damage every so seconds after the player has taken damage for the first time.

Gravity Effects



For the Gravity effects on objects this was something i spent alot of time on as i saw it as a fun challenge to work on and it would add alot of "Game Feel to the game since its all based around space.So that this could be used on any object i created the base of this through C++ which definately was much more difficult than all blueprint but it was a good chance to learn about 3d physics as i hadn't had much time to work in this field before and i found it very intresting.

For this script it sets up the variable that the blueprints then uses to create the line trace between the island and the item below it to calculate the amount to float and if the player then adds force to the island going downward it can adjust for that.With the variable of hover force made available to change in the editor it then makes it easier to work with different sizes of objects such as small rocks

Small Stalks

Documentation

Small stalks is a text based, top down game where you play as a farmer and the way the main plants would grow would be depending on the decisions/dialogue the player conveys to the AI characters when talking to them.The plants can then grow better or worse depending these respones or general chat with characters around the game.My general focus as the only game developer was to work on the dialogue system for the game since it was the main bases of it, it then couldnt be hard coded as it needed to be easy for other people in the team to add lots of dialogue and repsonses easily.

Engine & Tools: Unreal engine 4, Rider 2019, Github, Mixamo

Small Stalks Playthrough

This game was created for a second year University project between a large group of students and myself, over a couple of weeks to create a working demo or game.My role alongside an indie developer was to create the dialogue system in ue4. However, our expectations of what we could get done were a lot higher than realistically possible, which has lead to the game being left unfinished but enough for the university submission since it was only a game prototype that was needed. My main focus on this project was the entire communication between the player and the AI which lets the player talk to different characters and the choices/responses affect the environment around it.

Pickup & Inventory System



Since the game needed more for the player to be doing than just talking to these characters all the time we implemented a pickup system so that the player can pick up items like a watering can, fill it up and uses it to keep the plants alive, and this would then become a system that the plants have a happiniess and water level which is visible on the screen and the player has to keep running around to make sure these levels don't drop to nothing and the plants then die.

For this i had to do alot of research into creating pickups that could then be placed into a inventory system and used on a button press, however the one side of the design of the game that worked well in my favour was that the player can only pick up one at a time, Which meant that it was more a case of changing a variable or system on the player so know what they are holding.So each item is then fed through a array on the player that then gets updated every time the player adds to or removes from the inventory. This then could be used for the visual inventory hud on the screen so the player can see what items are picked up. Another idea the gorup came up with was for items to wear away or become unusable so they would have to keep picking up these items, however there wasn't enough time in the project for this.
